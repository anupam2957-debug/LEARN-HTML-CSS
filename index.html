HTML · 01

The structure of every HTML page

HTML stands for HyperText Markup Language. It isn't a programming language — it doesn't calculate things. It's a marking language: you take plain text and wrap it in tags to tell the browser what each piece is: "this is a heading," "this is a paragraph," "this is a picture."

Almost every tag comes in a pair: an opening tag like <p> and a closing tag like </p> (note the forward slash). Whatever sits between them is the element's content.

Every HTML file starts the same way. The boilerplate below is the skeleton you'll copy into every single project.
● ● ●  index.html
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is my first page.</p>
  </body>
</html>
Result

Hello, world!

This is my first page.

TagWhat it means
<!DOCTYPE html>Tells the browser "this is a modern HTML5 page." Always first, no closing tag.
<html>The outermost wrapper — everything lives inside it.
<head>Invisible info for the browser: page title, linked CSS files, etc.
<body>Everything the visitor actually sees goes here.
HTML · 02

Text and headings

Headings go from <h1> (biggest, most important — use only once per page, usually the title) down to <h6> (smallest). Regular text goes inside <p> paragraph tags.

● ● ●  index.html
<h1>Main Title</h1>
<h2>Section Title</h2>
<p>
  Some text can be <b>bold</b>,
  some <i>italic</i>,
  and you can force a
  <br>
  line break mid-paragraph.
</p>
Result

Main Title

Section Title

Some text can be bold, some italic, and you can force a
line break mid-paragraph.

<br> is a self-closing tag — it has no content and no separate closing tag, since a line break doesn't wrap anything.

HTML · 03

Links and images

A link uses the <a> tag (anchor). The destination is set with the href attribute. An image uses <img>, which is self-closing and points to a file with src.

● ● ●  index.html
<a href="https://anthropic.com">
  Visit Anthropic
</a>

<img src="cat.jpg"
     alt="A sleeping cat"
     width="120">
Result

Visit Anthropic

🖼 "A sleeping cat"

The alt attribute describes the image in words. It shows up if the image fails to load, and it's read aloud by screen readers — never skip it.

HTML · 04

Lists

Use <ul> (unordered list) for bullet points, or <ol> (ordered list) for numbered steps. Either way, each item goes inside <li>.

● ● ●  index.html
<ul>
  <li>Milk</li>
  <li>Eggs</li>
</ul>

<ol>
  <li>Preheat oven</li>
  <li>Mix batter</li>
</ol>
Result
  • Milk
  • Eggs
  1. Preheat oven
  2. Mix batter
HTML · 05

Attributes

Attributes give a tag extra information. They live inside the opening tag, as name="value" pairs. You've already seen href, src, and alt — here are others you'll use constantly.

<p id="intro" class="highlight">
  Hover me
</p>

HTML · 06

Div, span, and semantic tags

<div> is a generic block-level box with no meaning of its own — it's a container you use to group things for styling or layout. <span> is the same idea but inline, for wrapping a small chunk of text inside a sentence.

Modern HTML also has semantic tags that describe the role of a section instead of just being a plain box: <header>, <nav>, <main>, <section>, <article>, <footer>. Using these instead of endless <div>s helps browsers, search engines, and screen readers understand your page.

● ● ●  index.html
<div class="card">
  <p>
    My favorite color is
    <span style="color:teal">teal</span>.
  </p>
</div>
Result

My favorite color is teal.

HTML · 07

A table is <table>, containing rows <tr>. Each row holds header cells <th> or data cells <td>.

HTML · 08

Forms

Forms collect input from visitors. Everything sits inside <form>. The most common input is <input>, whose type attribute decides what kind of control it is.

CSS · 01

What is CSS?

If HTML is the skeleton (what's on the page), CSS — Cascading Style Sheets — is the styling: colors, fonts, spacing, layout. Every CSS rule has the same shape:

● ● ●  style rule
selector {
  property: value;
  property: value;
}

/* real example */
p {
  color: teal;
  font-size: 18px;
}
Result

Every <p> on the page turns teal and grows to 18px.

Three ways to add CSS to a page

MethodHow
Inline<p style="color:red"> — styles one single element directly.
InternalA <style> block inside <head> — styles this one page.
External (best practice)A separate .css file linked with <link rel="stylesheet" href="style.css"> — reused across many pages.
CSS · 02

Selectors: choosing what to style

A selector tells CSS which elements a rule applies to.

● ● ●  style.css
/* every element of this type */
h2 { color: #1F9C8A; }

/* only elements with this class */
.warning { color: #C0392B; }

/* only the one element with this id */
#logo { font-weight: bold; }
Result
h2 { color: teal }
.warning { color: red }
#logo { font-weight: bold }
SelectorMatches
pEvery <p> tag
.cardEvery element with class="card" (dot prefix)
#logoThe one element with id="logo" (hash prefix)
div pEvery <p> inside a <div>
*Everything on the page

CSS · 04

The box model

Every single HTML element is secretly a rectangular box made of four layers, from the inside out: content, padding (space around the content, inside the border), border, and margin (space outside the border, pushing other elements away). Drag the sliders and watch the box change live.

CSS · 05

Flexbox: lining things up

Set display: flex on a container and its direct children line up in a row (by default) and become easy to align, space out, or center — no more guessing with margins.

● ● ●  style.css
.row {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 10px;
}

CSS · 06

Positioning

The position property changes how an element is placed relative to the normal flow of the page.

ValueBehavior
staticDefault — sits in normal document flow.
relativeStays in flow, but can be nudged with top/left relative to where it'd normally sit.
absoluteRemoved from flow, positioned relative to the nearest positioned ancestor.
fixedPositioned relative to the browser window; stays put when you scroll.
● ● ●  style.css
.card { position: relative; }
.badge {
  position: absolute;
  top: -8px;
  right: -8px;
}

CSS · 07

Responsive design with media queries

A @media rule applies CSS only when a condition is true — most commonly, screen width. This is how a page rearranges itself for phones vs. desktops. Try resizing this browser window: this entire site's layout is responsive using exactly this technique.

● ● ●  style.css
.row { display: flex; }

/* below 600px wide screens */
@media (max-width: 600px) {
  .row {
    flex-direction: column;
  }
}
Result

Wide screen → items sit in a row. Narrow screen (under 600px) → the same items stack in a column automatically.
